Instructions (version 4)

Duration: 50 min + 5 de tolerance

We want to implement a software to help an IT company that provides ITConsultant services with the management of their employees. In this challenge, there exist two types of employees: those who pertain to the human resources management area, and the IT (information technology) experts, who pertain to ITConsultant.

A company is characterized by its name. A company can have multiple employees, but each employee only the belongs to a single company.

A employee is identified by its id (int), name (String) and monthly salary (int). The human resource employees do not have any extra attribute, however, the IT employees are associated their respective hourly rate (int), as well as the name of the client (String) to which they are allocated.

Tasks

  1. Create the package org.dropproject.samples.itcompanyassignment. All code for this mini-test must be created within this package.
  2. Analyse, in terms of inheritance, which relation exists between the described types of employees, knowing that the classes Employee, HRWorker e ITConsultant must be created;
  3. The super class should contain a constructor with all the attributes identified for employees, namely id (int), name (String) and salary (int). However, it should not be possible to instantiate an object using the super class's constructor;
  4. The human resources worked type is represented by the HRWorker class, and it does no have any attributes besides the ones defined for the Employee class.
  5. Add to your HRWorker class the following method:
    • A public String toString() method, which must be implemented to return the following:
      • "<id> | <name> | <salary> (HR);

      Example with human resources employee: 
      1 | Carolina Feliz | 1300 (HR)
    6. Add, to the appropriate classes, the following methods:
    • A public int getId() method, which returns the employee's ID.
    • A public String getName() method, which returns the employee's name.
    • A public int getSalary() method, which returns the employee's salary.
    • A public int getValue() method, which returns the IT employee's hourly rate.
    • A public String getClient() method, which returns the name of the client to which the IT employee is allocated.
  6. The ITConsultant class, wich represents the IT Consultant, must have a constructor, all the attributes defined for a employee, as well as the specific attributes value (int) and client (String).
  7. Add to your ITConsultant class the following methods:
    • A public String toString() method, which must be implemented to return the following:
      • "<id> | <name> | <salary> (IT Consultant)\n <client> | <value> (hourly value)";

      Example of employee of type ITConsultant: 
      1 | Diogo Costa | 1800 (IT Consultant)
Daimler | 26 (hourly value)
  8. Create the Company class, with the name attribute (String). Don't forget that the company has employees. The Company class, besides the constructor defined by its attribute, must have the followind methods:
    • A public String getName() method, which must return the companies' name.
    • A public ArrayList<Employee> getEmployees() method, which must return all the companies' employees.
    • A public void addEmployee(Employee employee) method, which allows adding a employee to the company.
    • A public Employee getEmployeeHighestHourlyRate() method, which returns the employee with the highest hourly rate. If there is a tie (i.e. more than one employee with the same maximum hourly rate value), the algorithm must consider the employee whose name is alphabetically smaller (e.g. since "A" is alphabetically lower than "I", when "Alf" and "Inete" have the same hourly rate, "Alf" must be returned);
    • A public String toString() method, which must be implemented to return the following:
      • "The company <name> does not have employees." - if the company does not have any employee;
      • "The company <name> has <number_employees> employees:" - if the company has at least one employee. Besides that, the information for each employee should be display, considering the respective type:
      • "<id> | <name> | <salary> (HR)" - for human resource employees;
      • "<id> | <name> | <salary> (IT Consultant)\n <client> | <value> (hourly value)" - for IT employees;

      Example of a company with 3 employees (one HR employee and two IT consultants): 
      The company Clever Smart Reloaded Consulting has 3 employees:
1 | Ana Bela | 1200 (HR)
2 | João Mário | 1500 (IT Consultant)
Nova Base | 22 (hourly value)
3 | Ana Falcão | 1670 (IT Consultant)
Opensoft | 24 (hourly value)
  9. You must create the Main class, which must contain the following methods:
    • The main(...) method, without behaviour (empty).
    • A public static Company myCompany() method, which must return an object of class Company with the following data:
      • Company name: the last 4 digits of your student ID.

Attention

  • Be careful to respect the indicated data types and formats.
  • It is mandatory to respect the return syntaxes of the toString() methods.
  • It may be necessary to implement abstract methods that allow differentiating the various types of employees.
